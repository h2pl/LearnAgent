## 概述

[上一篇](./01-llm-overview.md)我们知道了 LLM 是什么、它的核心原理是"预测下一个词"。但 LLM 不是凭空出现的——在它之前，人类已经花了几十年教机器"读懂文字"。

这篇文章讲的就是这段前史：从最早的手写规则，到统计学习，到深度学习，再到 Transformer 架构横空出世。理解这段演进，你才能真正明白 LLM 为什么是今天这个样子，以及为什么 Transformer 如此重要。

## 什么是 NLP

**NLP（Natural Language Processing，自然语言处理）** 就是让计算机处理人类语言的技术。它并不遥远——搜索引擎能理解你的搜索词、垃圾邮件过滤器能识别广告、输入法能联想下一个词、翻译软件能把中文变成英文，背后都是 NLP。

传统 NLP 的发展大致经历了三个阶段，每个阶段都在用不同的方式解决同一个问题：**怎么让机器从文字中提取有用的信息？**

## 规则时代（1950s-1990s）

**核心思路：人工告诉机器怎么做。**

最早的做法是直接写 if-else 规则。举个例子：

```
# 伪代码：判断一句话是不是在问问题
if 句子以"吗"结尾 or 句子以"？"结尾:
    return "这是一个问题"
```

这种方式对简单场景有效，但语言太灵活了——"你去不去？""帮我看看这个""难道你觉得这样好"都是问句，规则根本写不完。最终规则库越来越庞大，越来越难维护。

## 统计学习时代（1990s-2013）

**核心思路：让机器从数据中学规律。**

既然规则写不完，不如让机器自己从大量文本中总结规律。核心思路变成了：人类设计"特征"（告诉机器从哪些角度看文本），然后让算法自动学习这些特征和结果之间的关系。

举个直觉的例子——做垃圾邮件过滤：

```
人类设计的特征：
  - 邮件里出现"免费"这个词几次？
  - 有没有出现"点击领取"？
  - 发件人在不在通讯录里？
  - 有没有附件？

算法学到的规律：
  "免费"出现 3 次以上 + 不在通讯录 → 90% 概率是垃圾邮件
```

这比纯规则灵活多了，但问题是**特征要靠人来设计**。做情感分析要想一套特征，做机器翻译又要想另一套。每个 NLP 任务都需要大量的领域经验来做特征工程。

## 深度学习时代（2013-2017）

**核心思路：让机器自己学特征。**

2013 年 Word2Vec 的出现是一个转折点。它用神经网络把每个词变成一组数字（向量），而且语义相近的词会被映射到相近的位置。比如"国王"和"女王"的向量很近，"国王"和"苹果"的向量很远。这样机器第一次有了对词义的"感觉"。

随后出现了能处理整句话的神经网络：

- **RNN / LSTM**：像人类阅读一样，从左到右逐词处理，记住前面的上下文。但它有个致命缺陷——句子太长就会"忘"掉前面的内容（就像你读一篇很长的文章，读到结尾已经忘了开头说什么）
- **Seq2Seq**：用于机器翻译等任务，一个网络负责"读"（编码器），一个网络负责"写"（解码器）。后来给解码器加了一个技巧——在生成每个词时回头看编码器的输出，关注最相关的部分。这个技巧为后来 Transformer 的核心机制埋下了种子

这个阶段的进步在于：**机器开始自动学习语言的特征，不再需要人工设计了。** 但每个任务仍然需要单独训练一个模型，做翻译的模型不会做摘要，做分类的模型不会做问答。

## Transformer 的诞生（2017）

**然后 Transformer 来了。**

2017 年 Google 提出了 Transformer 架构，它的核心思想其实很直觉：处理一个词的时候，不再像 RNN 那样只能看到前面的内容，而是**同时看到整个句子的所有词**，并自动判断哪些词跟当前词最相关。这个机制叫**自注意力（Self-Attention）**。

打个比方：RNN 像是逐字朗读一篇文章的人，读到后面就忘了开头；Transformer 像是能同时看到整篇文章的人，处理任何一个段落时都能回头参考所有其他段落。

这带来了两个关键优势：
- **不会遗忘**：无论句子多长，每个词都能直接关注到其他任意位置的词
- **可以并行**：所有词同时处理，训练速度比 RNN 快得多，能充分利用 GPU

Transformer 解决了前面所有方法的核心瓶颈，但它本身只是一个架构——就像发明了发动机，还不等于造出了汽车。从 Transformer 到今天的 LLM，中间还经历了预训练、规模涌现、人类对齐等一系列关键突破。
