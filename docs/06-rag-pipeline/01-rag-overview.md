# RAG 概述

> RAG（Retrieval-Augmented Generation）是给 LLM 加外部知识的核心能力——先检索相关文档，再让模型基于这些文档生成答案，解决 LLM 知识过时和幻觉问题。

你好，我是江小湖。在 [从零实现最小 Agent](../05-agent-loop/05-minimal-agent.md) 中，你实现了一个能调用工具的 Agent。但工具只能获取实时数据（如天气），无法访问私有知识库（如公司文档、个人笔记）。RAG 解决这个核心问题：**让 LLM 基于你的私有知识库回答问题**。

## LLM 的知识困境

LLM 有三个固有的知识局限：

| 局限 | 说明 | 后果 |
|------|------|------|
| **知识过时** | 训练数据截止到某个时间点 | 无法回答最近发生的事件 |
| **缺乏私有知识** | 只知道公开数据 | 无法回答公司内部问题 |
| **幻觉** | 生成看似合理但错误的内容 | 答案不可靠 |

**传统解决方案的局限**：

- **微调（Fine-tuning）**：将知识"烧"进模型参数，成本高、更新慢
- **Prompt Engineering**：上下文窗口有限，无法塞入大量文档
- **工具调用**：只能获取实时数据，无法访问历史文档

**RAG 的优势**：不修改模型，而是**在推理时动态检索相关文档**，让模型基于检索结果生成答案。成本低、更新快、可扩展。

## RAG 的核心思想

RAG（Retrieval-Augmented Generation，检索增强生成）的核心思想是**先检索，再生成**：

```
用户提问 → 检索相关文档 → 将文档作为上下文 → LLM 基于上下文生成答案
```

**类比**：RAG 就像开卷考试——你不需要记住所有知识，只需要知道去哪里找答案。

**RAG 的三个关键步骤**：

| 步骤 | 说明 | 输入 | 输出 |
|------|------|------|------|
| **索引（Indexing）** | 将文档切分并向量化 | 原始文档 | 向量数据库 |
| **检索（Retrieval）** | 根据问题检索相关文档 | 用户问题 | 相关文档片段 |
| **生成（Generation）** | 基于文档生成答案 | 问题 + 文档片段 | 最终答案 |

## RAG 与微调的区别

| 维度 | RAG | 微调 |
|------|-----|------|
| **知识来源** | 外部文档 | 训练数据 |
| **更新成本** | 低（重新索引） | 高（重新训练） |
| **可解释性** | 高（可引用来源） | 低（黑箱） |
| **适用场景** | 知识密集型问答 | 风格/格式调整 |
| **延迟** | 较高（检索+生成） | 较低（仅生成） |

**选择建议**：

- **知识频繁更新** → 用 RAG（更新文档即可）
- **需要引用来源** → 用 RAG（可追溯到具体文档）
- **调整模型风格** → 用微调（如让回答更专业/更口语）
- **两者结合** → 微调调整风格 + RAG 提供知识

## RAG 的典型应用场景

| 场景 | 说明 | 价值 |
|------|------|------|
| **企业知识库** | 员工查询公司制度、产品文档 | 避免重复解答，提高效率 |
| **客服系统** | 基于产品手册回答用户问题 | 答案准确，可追溯来源 |
| **个人助手** | 查询个人笔记、收藏的文章 | 私有知识，个性化回答 |
| **代码助手** | 查询代码库、API 文档 | 精准定位，减少幻觉 |
| **法律/医疗** | 基于法规/指南回答专业问题 | 降低风险，提高合规性 |

**RAG 的核心价值**：将 LLM 的"通用知识"与你的"私有知识"结合，让答案既准确又可靠。

## RAG 的完整流程

RAG 的完整流程包含三个阶段，每个阶段都有多种策略选择：

```
文档 → 切分 → 向量化 → 存入向量数据库
                ↓
用户问题 → 向量化 → 相似度检索 → 获取相关文档
                ↓
问题 + 文档 → LLM → 生成答案
```

**索引阶段**：

1. **文档解析**：将 PDF/Word/HTML 等格式转为纯文本
2. **文档切分**：按固定长度/语义边界切分为片段
3. **向量化**：用 Embedding 模型将文本转为向量
4. **存储**：将向量存入向量数据库

**检索阶段**：

1. **问题向量化**：用同样的 Embedding 模型将问题转为向量
2. **相似度检索**：在向量数据库中查找最相似的文档片段
3. **重排序**（可选）：对检索结果进行二次排序，提高相关性

**生成阶段**：

1. **构造 Prompt**：将问题和检索结果组合成 Prompt
2. **LLM 生成**：让模型基于上下文生成答案
3. **后处理**（可选）：格式化答案、添加引用来源

## 总结

- **RAG** 是"先检索，再生成"的范式，解决 LLM 知识过时和幻觉问题
- **核心价值**：将通用知识与私有知识结合，让答案既准确又可靠
- **三个阶段**：索引（文档→向量）、检索（问题→文档）、生成（文档+问题→答案）
- **与微调的区别**：RAG 更新成本低、可解释性高，适合知识密集型场景
