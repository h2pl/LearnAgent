# 01 — 多模态基础：认知与机制

本章是多模态栏目的起点，帮助你建立完整的认知框架和技术基础。从"多模态是什么"出发，逐步深入到编码、融合、对齐三大核心机制。

## 学习路径

本章 8 篇文章分为三个台阶，逐级深入。

### 第一台阶：建立认知（01-05）

适合所有开发者，建立对多模态的完整认知。

[什么是多模态](./01-什么是多模态.md)：用生活例子引出多模态的概念、为什么直到最近才成熟。

[为什么需要多模态](./02-为什么需要多模态.md)：从具体场景出发，对比流水线方案和原生方案的优劣。

[模态异质性](./03-模态异质性.md)：不同模态的信息存在于完全不同的表示空间中——这是多模态技术的核心挑战。

[两条技术路线](./04-两条技术路线.md)：理解侧（Transformer）和生成侧（扩散模型）是两个完全不同的世界。

[2026 模型全景与选型](./05-模型全景与选型.md)：主流模型的能力对比和场景选型指南。

### 第二台阶：核心机制（06-08）

需要一定的技术基础，深入理解多模态模型的底层工作原理。

[编码](./06-编码.md)：从像素到向量——各模态的编码方式和实践影响。

[融合](./07-融合.md)：三种融合策略——晚期融合、交叉注意力、早期融合。

[对齐与表示学习](./08-对齐与表示学习.md)：CLIP 对比学习、跨模态嵌入空间、质量评估。
