# 01 — 多模态基础：认知与机制

本章是多模态栏目的起点，帮助你建立完整的认知框架和技术基础。从"多模态是什么"出发，逐步深入到编码、融合、对齐三大核心机制。

## 学习路径

本章 10 篇文章分为三个台阶，逐级深入。

### 第一台阶：建立认知（01-05）

适合所有开发者，建立对多模态的完整认知。

[什么是多模态](./01-what-is-multimodal.md)：用生活例子引出多模态的概念、为什么直到最近才成熟。

[为什么需要多模态](./02-why-multimodal.md)：从具体场景出发，对比流水线方案和原生方案的优劣。

[模态异质性](./03-modality-heterogeneity.md)：不同模态的信息存在于完全不同的表示空间中——这是多模态技术的核心挑战。

[两条技术路线](./04-two-tech-routes.md)：理解侧（Transformer）和生成侧（扩散模型）是两个完全不同的世界。

[2026 模型全景与选型](./05-model-landscape.md)：主流模型的能力对比和场景选型指南。

### 第二台阶：核心机制（06-10）

需要一定的技术基础，深入理解多模态模型的底层工作原理。这部分按 Baltrušaitis 等人（2017）的五挑战 taxonomy 组织。

[表示学习](./06-representation.md)：联合与协调表示——各模态如何编码为统一向量空间。

[翻译](./07-translation.md)：示例与生成方法——如何在模态之间映射和生成内容。

[对齐](./08-alignment.md)：显式与隐式对齐——如何建立跨模态子元素之间的精确对应关系。

[融合](./09-fusion.md)：模型无关与模型驱动——如何合并多模态信息做统一预测。

[协同学习](./10-co-learning.md)：并行、非并行与混合数据——如何用富模态帮助贫模态。
