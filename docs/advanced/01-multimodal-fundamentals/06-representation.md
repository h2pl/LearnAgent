# 表示学习：各模态如何变成统一向量

多模态处理的第一步，是把不同模态的原始数据转换成模型能计算的向量表示。Baltrušaitis 等人（2017）在综述中将多模态表示学习列为首要挑战——好的表示是后续所有任务的基础。

## 单模态编码

每个模态的数据天然存在于不同的表示空间中。文本是离散符号序列，图像是像素网格，音频是时域波形。把它们统一成向量，是表示学习的第一层。

**文本编码。** Tokenizer 把句子切成 token，每个 token 从 embedding 表查到一个向量。100 个词的文本 ≈ 100 个 token，信息密度最高——每个向量都在承载语义。但文本的表达能力有天花板：你没法用文字精确描述一张脸上的微表情或一段音乐中的情绪转折。

**图像编码。** 主流做法是 Vision Transformer（ViT）把图片切成小块（patch），每个 patch 独立编码。一张 1024×1024 的图片切成 16×16 网格 = 256 个 patch。这就是多模态 API 定价的逻辑：一张图 ≈ 256 个 token，GPT-5.5 的输入价 $2.50/1M token，一张图 ≈ $0.0006。缩分辨率是最有效的省钱手段——patch 变少了。

**音频编码。** 两种主流方案：频谱图方案把波形转成梅尔频谱图，用类似图像的方式处理；Whisper 方案直接把音频波形编码成向量序列。核心挑战是序列长度爆炸——10 秒语音产生数千个时间帧。

**视频编码。** 负担最大的模态。1 分钟 1080p 视频（30fps）= 1800 帧。实践中靠帧采样策略控制成本：均匀采样、关键帧提取、自适应采样。

## 多模态表示的两条路径

单模态编码解决的是"每个模态怎么变成向量"。多模态表示学习解决的是"不同模态的向量怎么组合在一起"。Baltrušaitis 将已有方法分为两类：联合表示和协调表示。

### 联合表示

联合表示把多个模态投影到同一个表示空间：

```
𝐱_𝑚 = 𝑓(𝐱_1, 𝐱_2, ..., 𝐱_𝑛)
```

适合训练和推理时所有模态都存在的场景。

**神经网络并置。** 最简单的是早期融合——把各模态特征向量直接拼接。更先进的做法是每个模态先经过几层独立网络，再通过一个隐藏层投影到联合空间，之后可以继续堆叠隐藏层做预测。GPT-5.5 和 Gemini 3.1 Pro 的架构本质上是这种思路的极端版本——文本 token 和图像 patch token 在同一个 Transformer 里做注意力，模型的每一层都在同时处理所有模态。

**图模型。** 深度玻尔兹曼机（DBM）通过堆叠受限玻尔兹曼机来学习多模态联合表示。优势是无监督训练、天然处理缺失模态、可以从表示生成样本。劣势是训练计算成本高。Srivastava 和 Salakhutdinov（2012）提出的多模态 DBM 在 Alzheimer 分类等医学任务上有成功应用。

**序列表示。** 对于变长序列（句子、视频、音频流），RNN 和 LSTM 可以把整个序列编码成一个隐藏状态。多视角 LSTM（MV-LSTM）在每步显式建模模内和跨模交互，适合处理时序多模态数据。

### 协调表示

协调表示不把模态投影到同一空间，而是分别处理每个模态再通过约束让它们的表示空间"对齐"：

```
𝑓(𝐱_1) ∼ 𝑔(𝐱_2)
```

适合推理时只有部分模态存在的场景，比如跨模态检索——你给文字搜图片，文本编码器和图像编码器各自独立运行，但它们的输出空间是协调的，可以直接算相似度。

**相似度模型。** 最小化匹配对之间的距离、最大化不匹配对的距离。CLIP 是典型代表：文本编码器和图像编码器分别把输入映射到同一 embedding 空间，用对比损失拉近匹配对。DeViSE（deep visual-semantic embedding）和 Kiros 等人的图文协调表示都用了这个思路。训练完成后，"猫的图片"和"a cat"在向量空间里相近——这就是搜索语义的底层原理。

**结构化空间。** 在相似度的基础上施加更多约束。跨模态哈希要求表示同时满足：二进制编码、同物同码、保持相似性。CCA（典型相关分析）最大化两个模态之间的相关性，保持新空间正交。KCCA 用核方法做非线性扩展，但扩展性差；DCCA（深度 CCA）解决了这个问题。Deep canonically correlated autoencoders 在相关性的基础上加入重建项，保留模态独有信息。

## 产品级权衡

实际产品中，表示学习的选择决定了架构弹性：

| 产品 | 表示方式 | 为什么选这个 |
|------|---------|------------|
| CLIP | 协调表示（对比） | 文本和图像编码器独立，换编码器不影响对方 |
| GPT-5.5 | 联合表示（Transformer） | 统一处理多模态，跨模态交互充分 |
| DeepSeek-V4 | 联合表示 + 专家路由 | 不同模态走不同专家路径，计算效率高 |
| Gemini 3.1 Pro | 联合表示（模态原生） | 1M 上下文预训练时就联合学习，无独立投影层 |
| DALL-E 3 | 协调表示（文本→图像） | 文本编码器（T5-XXL）和扩散模型分别优化 |

联合表示的优势是跨模态交互充分，劣势是必须所有模态同时输入、计算量随模态数指数增长。协调表示的优势是编码器独立灵活、适合检索类任务，劣势是跨模态信息交互有限、丢失低层特征关联。
