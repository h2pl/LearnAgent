# 模态异质性：不同模态的本质差异

多模态听起来简单——不就是"支持更多输入类型"吗？但不同模态的信息天然存在于完全不同的表示空间里，这种差异叫做**模态异质性（Modality Heterogeneity）**。

把一段文字、一张图片、一段语音分别用"数字"表示出来，你会发现它们的结构完全不同。

## 四种模态的表示差异

### 文本：离散符号序列

文本是最"干净"的模态。每个词或子词对应一个 token，token 之间通过嵌入向量（embedding）映射到连续的向量空间。一段 100 个词的文本，变成 100 个向量组成的序列。

信息密度高：每个 token 都承载了明确的语义。但文本的问题是缺乏感官信息——你没法从文字描述中还原说话人的语气，也看不到一张图片的空间布局。

### 图像：连续的像素矩阵

一张 224×224 的图片，是一个 224×224×3（RGB 三通道）的像素矩阵。每个像素是 0-255 的数值，表示亮度或颜色强度。

和文本相比，图像的表示有两个特点：
- **空间结构重要**：像素之间的位置关系就是信息本身。一张猫的照片里，"猫在图片的中央偏左"这种空间关系，需要模型从像素的排列方式中提取。
- **信息冗余大**：196 个 patch 中很多是背景，不承载关键信息。一张蓝天白云的图片，一大半像素都在描述"蓝色"。

### 音频：时间序列波形

音频的表示又不一样。一段 16kHz 采样率的语音，每秒产生 16000 个采样点。每个采样点是一个浮点数，表示该时刻的声压。

音频的关键特征在时间维度上展开：音高（基频）、音色（频谱分布）、节奏（时间间隔）、语速（时长变化）。常见的做法是把原始波形转成梅尔频谱图（Mel Spectrogram），把时间-频率信息变成一张"图像"来处理。

### 视频：空间 + 时间

视频是四种模态中最复杂的。一段 1 分钟、30fps 的视频，包含 1800 帧图像，外加帧与帧之间的时序关系。视频不仅是"很多张图片"——运动模式、事件序列、物体在时间上的连续变化，这些时序信息让视频理解的复杂度比图像高一个数量级。

## 为什么异质性是个问题

不同模态的表示差异，带来三个直接的技术挑战：

**维度不匹配**。一段文本可能只有几百个 token，但一张图片经过 patch 编码后产生几百个向量，一段语音转成频谱图后产生数千个时间帧。这些不同维度的表示需要被模型统一处理。

**信息密度不同**。文本 token 语义密度高，但缺失感官细节；图像包含丰富的视觉信息，但有大量冗余。模型需要识别哪些信息重要、哪些是噪声。

**语义鸿沟**。"猫"这个文字和"一张带毛的、会动的、有胡须的生物"的图片——两者描述的是同一个概念，但表示形式完全不同。让模型在这两种表示之间建立准确的对应关系，是跨模态对齐的核心难题。

## 类比：三个不同的地图

想象你要描述你所在的城市：

- **文本模态**就像给你的朋友写一段文字描述——"从地铁站出来左转，走 200 米看到红色大楼"
- **图像模态**就像拍一张街景照片——"一看就知道是这里"
- **语音模态**就像用语音发一段导航——音调、语速、停顿都承载了"这个地方很复杂"的额外信息

三种方式描述的是同一个地方，但信息的呈现方式完全不同。多模态模型需要同时理解这三种描述，并能说清"照片里的红房子就是文字描述中的红色大楼"——这就是模态异质性带来的挑战。

## 模态组合速查

不是所有模态组合都已经成熟到可以上生产。截至 2026 年中：

| 组合 | 成熟度 | 推荐场景 |
|------|--------|----------|
| 文本 + 图像 → 文本 | ✅ 成熟 | 文档分析、视觉问答 |
| 语音 → 文本 | ✅ 成熟 | 语音识别 |
| 文本 → 图像 | ✅ 成熟 | 图像生成 |
| 文本 + 音频 → 文本 + 音频 | ⚠️ 可用 | 实时语音对话 |
| 文本 + 图像 → 动作 | ⚠️ 可用 | 屏幕控制 |
| 文本 + 图像 + 音频 → 综合推理 | 🔄 前沿 | 多模态 Agent |
| 实时视频流理解 | 🔄 早期 | 监控、直播分析 |
