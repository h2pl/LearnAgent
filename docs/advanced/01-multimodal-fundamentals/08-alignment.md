# 跨模态对齐：建立子元素间的精确对应

翻译把整个模态 A 变成整个模态 B。但很多场景下，你需要知道"A 中的哪个部分对应 B 中的哪个部分"——这就是对齐（Alignment）要解决的问题：找到不同模态子元素之间的直接对应关系。

Baltrušaitis 将对齐分为显式对齐和隐式对齐两类。

## 显式对齐

显式对齐把"对齐"本身作为建模目标——模型需要明确输出"第 X 个视频帧对应第 Y 句话"这类对应关系。

**无监督对齐。** 不需要标注的对应关系，依靠结构假设来推断对齐。最经典的算法是动态时间规整（DTW）——一种动态规划方法，通过时间轴弯折找到两个序列之间的最优匹配。DTW 已经在音视频对齐（语音到文本、视频到剧本）中广泛应用。Tapaswi 等人（2015）定义了视觉场景和句子之间的角色外貌相似度，在无标注条件下对齐电视剧与剧情摘要。CCA 被融入 DTW 形成联合对齐+映射的框架，Deep canonical time warping 进一步支持非线性对齐。

另一种无监督路径是用图模型。Cour 等人（2008）用生成式图模型对齐电影镜头与剧本。Malmaud 等人（2014）用因子化 HMM 对齐烹饪步骤食谱与制作视频。Naim 等人用层级 HMM 对齐句子与视频帧，并扩展至动词-动作对齐。

**监督对齐。** 有标注的对齐数据驱动模型学习相似度度量。Bojanowski 等人（2013）扩展典型时间规整以利用弱监督对齐信号。Mao 等人（2016）用 LSTM 语言模型 + CNN 视觉模型为"引用表达式"（referring expression）评估图文匹配质量——判断"左边那只橘猫"这句话精确对应图片中的哪块区域。Zhu 等人（2015）用 CNN 训练场景-文本相似度模型对齐电影与对应的书籍章节。

## 隐式对齐

隐式对齐不显式输出对应关系，而是在另一个任务（翻译、问答）训练过程中隐式学习对齐作为中间步骤。

**注意力机制。** 这是隐式对齐最流行也最成功的实现方式。标准的编码器-解码器把整个源压缩成一个向量再做解码；注意力机制则允许解码器每一步"关注"源的不同子部分。生成图片描述时，生成"cat"这个词的步骤会关注图片中包含猫的区域——这种对齐不是在标注数据中学的，而是模型在训练中自己发现的。

Xu 等人（2015）在图像描述中首次将注意力可视化，展示了模型在生成每个词时关注的图像区域。Bahdanau 等人（2014）在机器翻译中提出注意力，解决了长句翻译中编码器瓶颈的问题。自此注意力成为多模态对齐的标准工具：

- **图像描述。** 生成每个词时关注图片的不同区域
- **视频描述。** 生成描述动作的词语时关注对应的视频帧段
- **语音识别。** 生成文字时关注音频的不同时间窗
- **VQA。** 回答问题关注图片中的相关区域和问题中的相关词

**相似度隐式对齐。** Karpathy 等人（2014）提出了另一种思路：学习句子片段和图像区域之间的点积相似度。模型被训练来做检索，而非直接做对齐——但训练过程中隐式地学会了"这个短语对应那块区域"。这就是 CLIP 类模型的基础思路：不做显式对齐标注，但对比学习迫使编码器在表示空间中隐含地对齐概念。

## 产品级对齐策略

| 产品 | 对齐方式 | 对齐粒度 | 为什么这么选 |
|------|---------|---------|------------|
| CLIP | 隐式（对比学习） | 概念级 | 不需要标注数据，4 亿对数据规模驱动 |
| DALL-E 3 | 隐式（生成式） | 概念+属性级 | 文本跟随性要求高，生成式对齐更精细 |
| GPT-5.5 | 隐式（注意力） | token 级 | 注意力让解码器精确定位输入中的相关信息 |
| Sora 2 | 隐式（注意力+DiT） | 时空级 | 视频需要帧间对齐，注意力在时空两个维度展开 |
| Runway Gen-4.5 | 隐式（跨帧注意力） | 时空级 | Director Mode 跨镜头保持物体身份对齐 |

CLIP 的对齐精度在概念级有效——"猫"对"猫"，但学不到"胡须有几根"这种像素级对应。DALL-E 3 的生成式对齐更进一步：文本编码器主动调整输出分布配合扩散模型，实现对 Prompt 属性的更精准跟随。但无论哪种方式，对齐精度最终受限于信息差——一张 1024×1024 的图片含约 3MB 像素信息，"a cat on the windowsill" 只有 5 个 token，99.9% 的像素细节只能靠模型"脑补"。
