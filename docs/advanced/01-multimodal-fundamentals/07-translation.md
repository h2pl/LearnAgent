# 跨模态翻译：生成图片、视频与音乐

多模态翻译（Translation）解决的是：给定一个模态的实体，生成它在另一个模态中的对应实体。文生图、文生视频、语音合成、图生文——所有这些应用的底层都是翻译问题。

Baltrušaitis 将翻译方法分为两类：基于示例的和生成式的。

## 基于示例的翻译

示例方法依赖一个"字典"（训练数据中的配对样本）。翻译就是去字典里找最合适的已有答案。

**直接检索。** 给定一个查询（如图片），在字典里找到最相似样本的翻译结果直接拿来用。实现简单，但效果取决于字典覆盖率和相似度度量的质量。Ordóñez 等人（2010）用全局图像特征检索候选描述文本，再通过重排筛选最优结果。Devlin 等人（2015）证明简单的 kNN 检索加上共识选择机制就能与复杂生成模型竞争。

**语义空间检索。** 不再在单模态空间里比相似度，而是把查询和字典都投影到共享语义空间再检索。Farhadi 等人（2010）构建了 \<object, action, scene\> 三元组语义空间。Socher 等人（2014）和 Karpathy 等人（2014）学习深度协调表示用于双向检索——既可以从文本搜图片，也可以从图片搜文本。优势是跨模态匹配更准确，且天然支持双向翻译。

**组合式翻译。** 检索多个相关示例再组合成新答案。Kuznetsova 等人（2014）先检索描述视觉相似图片的短语，再用整数线性规划组合成新描述。Gupta 等人（2012）找到最相似的 k 张图片，从它们的描述中提取短语组合成目标句子。比纯检索灵活，但组合规则依赖人工设计。

示例方法的根本局限：模型就是整个字典，容量大、推理慢。而且如果字典里没有合适的匹配，翻译质量会断崖下降。

## 生成式翻译

生成方法不是从字典里找答案，而是训练一个模型学会从源模态生成目标模态。挑战更大——不仅要理解源模态，还要能产生目标信号或符号序列。

**语法驱动生成。** 早期方法用预定义模板减化问题。Kojima 等人（2002）从视频中检测人体位置和动作，用规则生成行为描述。Guadarrama 等人（2013）预测 \<subject, verb, object\> 三元组，结合语义分层和语言模型生成描述。缺点受限于语法模板的表达能力。

**编码器-解码器。** 这是当今翻译任务的标准架构。编码器把源模态压缩成隐表示，解码器基于隐表示生成目标模态。针对不同语言对（modality pairs）的变体：

- **文生图。** DALL-E 3 用 T5-XXL 编码文本为 4096 维向量，作为条件送入扩散模型的去噪过程。文本编码器和扩散模型联合训练——编码器要学会"这些向量是供去噪模型使用的"，扩散模型要学会"文本向量中的每一维都在提示某个视觉特征"。这解释了为什么 DALL-E 3 对 Prompt 的跟随性优于用 CLIP 的 Midjourney。
- **文生视频。** Sora 2 用 DiT 架构把文本编码和视频 patch 在统一空间中做扩散生成。视频的时序一致性要求编码器不仅要理解"什么物体"，还要理解"物体怎么运动"。Runway Gen-4.5 引入 Director Mode，通过跨帧注意力机制保持多镜头中物体的一致性。
- **文生音乐。** Suno V5 用 Transformer 编码歌词和风格描述，音频解码器生成完整波形。3 分钟 44.1kHz 音频包含约 1600 万个采样点，歌词和风格描述约 500 token，扩展比 1:32000。歌词中的 "a gentle piano melody" 被展开成数百万个具体音色采样。
- **语音合成。** ElevenLabs 用编码器-解码器从文本生成语音波形，支持语音克隆和情绪控制。关键挑战是韵律生成——同样的文本在不同语境下有不同的重音、语调和节奏。

**连续生成。** 源模态和目标模态都是连续流的场景，比如文本到语音的实时合成和视觉语音生成。这类方法需要处理流式输入、对齐时间轴、保持低延迟。GPT-5.5 的 Realtime API 和 Gemini Live 都支持音频流的连续生成，编码和生成几乎同时进行。

## 翻译的评估困局

生成式翻译面临的核心问题是评估。图像描述的候选答案空间极大——一张图可以有无数种"正确"的描述。BLEU、ROUGE、CIDEr 等自动指标与人工判断的相关性有限。VQA 把图像理解转化为问答来部分规避编码差异，但评估问题仍未彻底解决。

## 产品级权衡

| 产品 | 翻译方向 | 方法 | 关键差异 |
|------|---------|------|---------|
| DALL-E 3 | 文→图 | 生成式（T5 + 扩散） | 联合训练使文本跟随性最强 |
| Midjourney V7 | 文→图 | 生成式（CLIP + 扩散） | 可用任意文本编码器，迭代快 |
| Sora 2 | 文→视频 | 生成式（DiT） | 统一时空表示，时序一致性最优 |
| Suno V5 | 文→音乐 | 生成式（Transformer + 音频解码） | 风格一致性高，长序列生成稳定 |
| ElevenLabs | 文→语音 | 生成式（编码器-解码器） | 语音克隆 + 情绪控制 |
| CLIP | 双向 | 示例式（协调检索） | 无生成功能力，检索精度最高 |
