你好，我是江小湖。在前面的章节里，你聊的全是文本——怎么让模型理解文字、生成文字、调用工具。但真实世界的信息远不止文字：你给朋友发一段语音、拍一张菜单照片、看一段监控视频——这些都是不同"模态"的信息。

多模态 AI 就是让模型不只处理文字，还能同时理解和生成图像、语音、视频等多种类型的信息。这一篇，你从全局视角把多模态这件事看清楚。

## 为什么需要多模态

先看一个场景。

你正在用一个 Agent 帮你分析一份 PDF 合同。合同里有文字，也有图表、签名、盖章。如果你的 Agent 只能处理文本，它需要先用 OCR 把图片里的文字提取出来，再用另一个模型分析图表数据，最后把这些零散结果拼在一起交给 LLM。

这条流水线能跑通，但问题很明显：

- **信息丢失**——OCR 提取不了手写批注的含义，图表的语义在转成文字时会失真
- **管线脆弱**——多个服务串起来，任何一个环节出错整个流程就断
- **上下文断裂**——模型看不到图片里文字和图表的空间关系，只能靠拼接后的文字猜

多模态模型解决这个问题方式更直接：**把所有模态的信息一起喂给模型，让它在内部完成跨模态理解。** 一张合同图片直接传给模型，它同时"看到"文字内容、图表数据、签名位置，然后给出综合分析。

这就是多模态的核心价值：**不是把多个单模态工具拼起来，而是让模型原生地理解多种信息之间的关系。**

人类就是这么认识世界的——你看到一个苹果，同时闻到它的气味、摸到它的质地，大脑自动把这些信息融合成"一个苹果"的概念。多模态 AI 想做的，就是这个过程的数字版本。

## 模态的异质性：不只是一个 API 的事

多模态听起来简单——不就是"支持更多输入类型"吗？但背后的技术挑战比想象中大得多。

CMU 的多模态机器学习课程（课程 11-777）总结了三个核心原则，其中最重要的一个叫**模态异质性（Modality Heterogeneity）**：

不同模态的信息，天然存在于完全不同的"表示空间"里。

具体来说：
- **文本**是离散符号序列，模型通过 tokenization 把它变成数字向量
- **图像**是连续的像素矩阵，包含空间结构、颜色、纹理等丰富信息
- **音频**是时间序列波形，包含音高、音色、节奏、语速等维度
- **视频**同时包含空间（每一帧的画面）和时间（帧与帧之间的变化）信息

这些模态之间的差异不是"格式不同"那么简单——它们承载信息的方式、密度、结构完全不同。一段 10 秒的语音可能包含 200 个词的内容，但同时携带了说话人的情绪、语气、口音等文本完全无法表达的信息。

这就是为什么多模态不是"加几个 API"就能解决的事。模型通过训练建立不同表示空间之间的桥梁，把"猫的图片"和"cat 这个词"联系起来——这涉及后面[第 2 篇](./02-core-mechanisms.md)要讲的编码、融合和对齐机制。

## 从拼接到原生：多模态的两次范式跃迁

理解多模态的发展脉络，能帮你判断哪些方案在今天还值得用。

### 第一阶段：流水线拼接（2024 年及之前）

早期的多模态方案本质上是把多个单模态模型串起来：

```
用户上传一张图片 → OCR 提取文字 → 视觉模型描述画面 → Whisper 转录语音
                                                    ↓
                                              LLM 综合所有文本结果
                                                    ↓
                                              TTS 生成语音回复
```

这个模式能工作，但代价很高：延迟大（串行调用 5+ 次 API）、成本高（每个服务都收费）、信息在转换过程中不断丢失（图片→文字→理解，中间损失了大量视觉细节）。

### 第二阶段：原生多模态（2025-2026）

从 GPT-4V（2023 年底）开始，到 GPT-4o（2024 年中）实现文本/图像/音频的实时多模态处理，再到 2025-2026 年各家跟进，多模态能力逐渐"内化"到单一模型中：

- **GPT-5.5**：文本 + 图像 + 音频 + 视频输入，单次 API 调用
- **Gemini 2.5 Pro**：文本 + 图像 + 音频 + 视频，支持原生音频输出（TTS），百万 token 上下文
- **Claude Opus 4.8**：文本 + 图像强项，结构化输出和工具调用领先，Arena 视觉榜第一

原生多模态的核心优势：**模型在内部就能做跨模态注意力计算**，不需要先把图像转成文字再理解。这让"看一张复杂图表然后回答问题"这种任务的质量和速度都有了质的飞跃。

### 判断标准

如果你现在还在用流水线拼接方案（OCR + ASR + Vision Model + LLM），大概率可以简化成一次原生多模态 API 调用。除非你的场景有特殊要求（比如需要处理某种模型不支持的模态），否则原生方案在质量、延迟、成本三个维度上都更优。

## 2026 模型全景：谁支持什么

截至 2026 年中，主流模型的多模态能力对比（模型数据来源：[llm-stats.com](https://llm-stats.com/)、[arena.ai](https://arena.ai/leaderboard)）：

| 梯队 | 模型 | 图像输入 | 音频输入 | 音频输出 | 视频输入 | 上下文长度 |
|------|------|----------|----------|----------|----------|-----------|
| Tier 1 旗舰全能 | Claude Opus 4.8 | 支持（强项） | 有限 | 不支持 | 有限 | 1M |
| | GPT-5.5 | 支持 | 支持 | 支持（Realtime API） | 支持 | 400K |
| | Gemini 2.5 Pro | 支持 | 原生支持 | 原生 TTS（Live API） | 支持 | 1M-2M |
| Tier 1.5 性价比 | Claude Opus 4.6 | 支持 | 有限 | 不支持 | 有限 | 200K |
| | GPT-5.4 | 支持 | 支持 | 支持（Realtime API） | 支持 | 256K |
| | GLM-5.1 | 支持 | 支持 | 支持 | 支持 | 1M |
| Tier 2 极致性价比 | DeepSeek V4-Pro | 支持 | 不支持 | 不支持 | 不支持 | 128K |
| | Qwen 3.7 Max | 支持 | 支持 | 不支持 | 支持 | 256K |
| | Kimi K2.6 | 支持 | 支持 | 支持 | 支持 | 1M |
| | MiniMax M3 | 支持 | 支持 | 支持 | 支持 | 1M |

**几个选型要点：**

- 需要**图像理解 + 结构化输出**：Claude Opus 4.8 是首选，Arena 视觉榜第一
- 需要**语音交互**（语音进语音出）：Gemini Live API 或 GPT-5.5 Realtime API
- 需要**视频理解**：Gemini 2.5 Pro 或 GPT-5.5，Kimi K2.6 也有不错的视频能力
- 需要**国产模型 / 更低成本**：DeepSeek V4-Pro（性价比之王）、Qwen 3.7 Max（推理特化）、Kimi K2.6（国产全能）
- 需要**超长上下文 + 多模态**：Gemini 2.5 Pro（1M-2M）、GLM-5.1 或 Kimi K2.6（1M）在长上下文场景也表现优秀

开源模型和闭源模型的差距在 2026 年已经大幅缩小，尤其在图像理解这个最成熟的模态上。但在跨模态推理（同时处理图像+文本+音频做综合判断）上，闭源前沿模型仍然有明显优势。

## 模态组合速查

不是所有模态组合都已经成熟。这里按"输入→输出"梳理一下当前可用性：

**已经成熟、可以直接上生产：**
- 文本 + 图像 → 文本回答（视觉问答、文档分析）
- 文本 → 图像（图像生成）
- 语音 → 文本（语音识别 / ASR）
- 文本 → 语音（语音合成 / TTS）

**2026 年已经可用，但需要做好评估：**
- 文本 + 音频 → 文本 + 音频（实时语音对话）
- 文本 + 图像 → 动作（屏幕控制 / Computer Use）
- 文本 → 视频（视频生成，质量仍在快速迭代）
- 文本 + 图像 + 音频 → 综合推理（多模态推理）

**仍在早期、谨慎用于生产：**
- 实时视频流理解 + 交互（延迟和稳定性问题）
- 触觉/传感器数据融合（主要用于机器人，通用场景少）
- 多模态生成（同时输出图像+语音+文本协调一致的内容）

## 什么时候该用多模态

不是所有场景都需要多模态。有时候纯文本反而更好。

**适合用多模态的场景：**
- 输入本身就是非文本的（用户发了一张图片、一段语音）
- 需要理解空间关系（图表、地图、UI 布局）
- 需要多通道信息互相印证（合同文字 + 盖章图片 + 签名比对）
- 交互方式需要更自然（语音比打字更方便的场景）

**不需要多模态的场景：**
- 信息已经是结构化的（JSON 数据、数据库记录）
- 只需要文本处理能力（翻译、摘要、代码生成）
- 延迟极其敏感（多模态处理必然比纯文本慢）
- 成本预算极紧（图像 token 成本远高于文本 token）

一个简单的判断方法：**如果你的输入去掉非文本信息后仍然完整，就不需要多模态。** 如果去掉后丢失了关键信息，那多模态就是必要的。

## 总结

多模态 AI 不是"文本模型的升级版"，而是让模型能像人一样同时处理多种信息的能力跃迁。2026 年，原生多模态已经成为主流——主流模型都支持图像输入，语音和视频能力也在快速成熟。
