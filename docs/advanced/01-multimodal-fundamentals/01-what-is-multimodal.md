# 什么是多模态

你每天处理的信息都是多模态的。刷朋友圈时看到一张美食照片（图像），点开后朋友发了一段语音说"这家超好吃"（音频），你回了一句"周末去试试"（文字）。整个过程自然流畅，你不会意识到自己在切换三种不同的信息类型。

AI 以前做不到这种切换。传统模型的"世界"只有文字——你给它一张图片，它看不懂；你发一段语音，它听不懂。多模态 AI 要解决的就是这个问题：让模型能同时处理和理解多种类型的信息。

## 什么是"模态"

模态（Modality）就是信息的呈现形式。常见的模态包括：

| 模态 | 例子 | 信息特征 |
|------|------|----------|
| 文本 | 文章、代码、对话记录 | 离散符号序列，人定义的抽象表示 |
| 图像 | 照片、截图、图表 | 连续的像素矩阵，包含空间结构和视觉特征 |
| 音频 | 语音、音乐、环境音 | 时间序列波形，包含音高、音色、节奏 |
| 视频 | 监控录像、教程视频 | 图像序列 + 时间维度，同时包含空间和时间信息 |

每种模态承载的信息类型不同，处理方式也不同。文本用 token 表示，图像用像素矩阵表示，音频用波形表示——这些表示方式的差异，就是多模态技术的核心挑战。

## 单模态模型 vs 多模态模型

你把一张发票照片发给一个纯文本模型，它不知道你在说什么。你发给一个多模态模型，它能读出发票号码、金额、日期，甚至能告诉你"这张发票的盖章位置在右下角"。

| 能力 | 单模态模型 | 多模态模型 |
|------|-----------|-----------|
| 输入类型 | 仅文本 | 文本 + 图像 + 音频 + 视频 |
| 理解图片 | ❌ 无法直接处理 | ✅ 能看图说话 |
| 理解语音 | ❌ 无法直接处理 | ✅ 能听写转文字 |
| 跨模态关联 | ❌ 不能 | ✅ 能建立图文对应关系 |
| 典型代表 | GPT-3.5、Llama 3 | GPT-5.5、Gemini 2.5 Pro、Claude Opus 4.8 |

多模态模型不是"文本模型加个图片功能"——它的架构设计从一开始就考虑了多种模态的联合处理。

## 一个直观的例子

你给模型发了一张餐厅菜单的照片，问："帮我看看哪些菜是素食？"

- **传统方案**：先用 OCR 把图片里的文字提取出来 → 得到纯文本菜单 → 让 LLM 分析哪些是素食。这个过程丢失了排版信息、价格标签的位置、菜品的配图——这些视觉信息可能是判断素食的线索。
- **多模态方案**：直接把菜单照片发给模型。它同时"看到"文字内容、菜品的排列方式、有没有标注"素食"图标，然后直接给出回答。

两种方案都能得出答案，但多模态方案更自然、信息损失更少、出错点更少。

## 为什么现在才成熟

多模态的概念十年前就有，但直到 2024-2026 年才真正进入生产阶段。三个关键转折点：

1. **Transformer 架构的泛化能力**：2017 年提出的 Transformer 不仅擅长处理文本，经过改造同样能处理图像（ViT）和音频。这意味着同一套架构可以覆盖多种模态。
2. **大规模多模态数据**：CLIP（2021）展示了用 4 亿图文对训练的可行性，证明了"数据够多，跨模态对齐就能做到"。
3. **统一 API 的成熟**：2025-2026 年，GPT-5.5、Gemini 2.5 Pro、Claude Opus 4.8 都把多模态能力集成到了单一 API 中，开发者不再需要拼凑多个服务。

这三个条件同时具备，多模态才从实验室走向生产。
