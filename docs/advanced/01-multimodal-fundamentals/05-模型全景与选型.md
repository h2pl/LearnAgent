## 按场景选型

### 图像理解 + 结构化输出

**首选：Claude Opus 4.8。**

Arena 视觉榜排名第一，在需要精确理解图像内容并输出结构化 JSON 的场景下表现最好。典型场景：文档分析、UI 截图解析、图表数据提取。

### 语音交互（语音进语音出）

**首选：Gemini Live API 或 GPT-5.5 Realtime API。**

两者都支持流式音频交互，延迟在亚秒级。Gemini 的优势是原生 TTS 集成（不需要单独调 TTS 模型），GPT-5.5 的优势是 Realtime API 的中断处理更成熟。

### 视频理解

**首选：Gemini 2.5 Pro。**

Gemini 的 1M-2M 上下文窗口在长视频场景下优势明显。GPT-5.5 和 Kimi K2.6 也有不错的视频能力，但上下文窗口限制了处理时长。

### 国产模型 / 更低成本

- **DeepSeek V4-Pro**：性价比之王，图像理解能力强，但不支持音频
- **Qwen 3.7 Max**：推理特化，适合需要强逻辑的多模态任务
- **Kimi K2.6**：国产全能型，1M 上下文，音频全链路支持
- **MiniMax M3**：全能均衡，适合通用场景

### 超长上下文 + 多模态

**首选：Gemini 2.5 Pro（1M-2M）。** GLM-5.1 和 Kimi K2.6（均 1M）在长上下文场景也表现优秀。

## 几个选型原则

1. **先用旗舰，再考虑降级。** 如果场景明确要求多模态深度推理，从 Tier 1 开始评估。只测试通过后才考虑 Tier 1.5 或 Tier 2 来降本。
2. **音频需求决定很多选择。** 如果需要语音进语音出，选择范围会缩小到 GPT-5.5、Gemini 2.5 Pro、Kimi K2.6、MiniMax M3 这几个。
3. **不要只看能力不看延迟。** 理解侧的推理时间是 0.5-3 秒，生成侧是 3-30 秒。如果延迟敏感，优先选择理解侧模型能力强的方案（比如用文本描述代替图片生成）。
4. **版本号敏感。** 模型版本迭代极快，选型时务必检查当前最新版本号和价格。
