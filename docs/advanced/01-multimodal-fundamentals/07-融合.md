# 融合：让模态对话

编码之后，每种模态都有自己的向量表示。但这些向量还在各自的空间里——"猫的图像向量"和"cat 的文字向量"之间没有直接联系。

融合（Fusion）就是**把这些不同来源的向量合到一起，让模型能做跨模态的理解**。

## 三种融合策略

不同模型采用不同的融合方式，从简单到复杂分别是：晚期融合、交叉注意力、早期融合。

### 晚期融合：最直接的方式

每种模态各自独立处理到最后一步，只把最终结果拼在一起做判断：

```
图像 → 视觉编码器 → 图像特征向量 
文本 → 语言编码器 → 文本特征向量 
                                  ↓
                            拼接 → 分类/回答
```

这是流水线方案的做法。优势是简单灵活，可以用现成的单模态模型拼在一起。劣势是模态之间的交互太晚，模型只能看到"图像说要下雨"和"文字说今天晴天"这种矛盾，但不知道为什么矛盾——它看不到图片里的哪块区域触发了"下雨"的判断。

**典型应用**：早期的多模态分类任务、简单的图文匹配。

### 交叉注意力：折中方案

每种模态保持自己的处理路径，但在关键层通过注意力机制"看"另一种模态的信息：

```
文本 Transformer 层 ←→ Cross-Attention ←→ 图像 Transformer 层
```

CLIP、Flamingo、LLaVA 等模型用了这种方式。文本模型和视觉模型各自独立编码，但在某些层通过交叉注意力让文本 token 可以"关注"到相关的图像 patch。

交叉注意力比晚期融合更精细——文本 token 可以知道"猫"这个词对应图片中的哪个区域。但因为各模态仍然保持独立处理路径，计算效率不如早期融合。

### 早期融合：最深度的交互

在最底层就把不同模态的向量拼在一起，让模型从头开始联合处理：

```
[图像 patch 向量] + [文本 token 向量] → 拼接成一个长序列 → 一起过 Transformer
```

这是 GPT-5.5、Gemini 2.5 Pro 这类原生多模态模型的做法。模态之间从最底层就能交互，文字 token 和图像 patch 在同一个 Transformer 中经过自注意力计算，彼此"看到"对方。

早期融合能实现最细粒度的跨模态关系建模——模型能编码"这个图像区域对应这段文字描述"这种对应关系。代价是训练成本高，需要大量多模态配对数据。

## 怎么判断你用的模型是哪种融合

| 模型 | 融合方式 | 表现特征 |
|------|---------|----------|
| GPT-5.5、Gemini 2.5 Pro | 早期融合 | 跨模态推理强，但训练成本高 |
| LLaVA、Qwen-VL | 交叉注意力 | 视觉编码器独立，通过投影层接入语言模型 |
| 流水线方案 | 晚期融合 | 各模态独立，结果拼接 |

融合方式直接影响模型的跨模态推理能力。早期融合的模型在需要深度跨模态理解的任务上表现最好——比如"图片里的猫在做什么，和这段文字描述一致吗"这种需要精细比对的问题。

## 对开发者的影响

- **能力上限**：如果你需要深度跨模态理解（医疗影像分析、多模态 Agent），早期融合的原生模型必不可少。
- **成本预算**：早期融合模型的多模态版本通常比纯文本版本贵。如果只是做简单的图文匹配，交叉注意力或晚期融合方案可能更经济。
- **延迟权衡**：交叉注意力需要在模态之间来回传递信息，延迟比晚期融合高。实时场景下需要评估这个影响。
