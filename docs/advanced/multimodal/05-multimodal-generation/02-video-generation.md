你好，我是江小湖。上一篇讲了图像生成——从噪声中一步步去噪得到一张图。视频生成是把这件事延续到时间轴上——每一帧不仅要好看，帧与帧之间还必须连贯。这难了不止一个数量级。

## 视频生成比图像生成难在哪

### 计算量爆炸

一张 1024×1024 的图片约 300 万个像素值需要生成。一段 5 秒 24fps 的视频 = 120 帧。如果每帧独立生成再拼接，就是 3.6 亿个像素值——还要确保帧间连贯。

实际工程中视频生成模型通过各种压缩和降采样来减轻负担，但即便高度优化后，一段 5 秒视频的生成仍然需要数分钟和大量 GPU 资源。

### 时序一致性

这是视频生成最大的技术瓶颈。静态图中你只需要画一只猫——视频中你需要画这只猫从卧到坐到跳到走，且每一帧的猫必须是**同一只猫**。

- 毛发纹理会变吗？眼睛颜色会变吗？体型的比例会漂移吗？
- 背景中物体的运动是否合理？一个杯子放在桌上，猫跳起时桌面震动了——但杯子还在原位，立马穿帮

时序一致性本质上是要求模型"记住"之前所有帧的信息，同时预测未来帧。这和图像生成的一次性独立出图是完全不同的问题。

### 物理合理性

视频中的运动需要符合物理直觉：
- 人走路不会突然出现在 3 米外
- 苹果从树上落下不会在中途悬浮
- 水面波纹应该从投石处向外扩散

当前的视频生成模型还没有真正学习物理——它们在有限的数据中看到了足够多"苹果从树上落下"的训练样本，总结出了外观模式，但没有学习 F=ma。当出现训练数据中少见的运动模式，物理合理性就崩溃。

### Sora 为什么贵：这个算力账

Sora 关停时 OpenAI 透露日均运营成本约 1500 万美元——按每天 10 万次生成计算，单次成本约 $150。对比 DALL-E 的单张图片成本约 $0.05，视频生成的单位成本是图像的 3000 倍。这不是技术做不到，是经济学上不成立。

## Sora 的故事：为什么最好的不一定能活下来

2024 年 2 月 OpenAI 发布 Sora 演示视频——一段 AI 生成的东京街头漫步，画面之逼真震惊了所有人。这是第一个让业内外共同认可"AI 视频生成能做到这个程度"的产品。

Sora 的核心技术是**扩散 Transformer（DiT）**——将扩散模型的去噪过程和 Transformer 的空间理解能力结合。它不止生成"看起来像视频"的内容，而是理解三维空间中的物体运动和光影变化——这是之前所有视频生成模型做不到的。

但 Sora 有两个致命问题：

**一问：技术先进 ≠ 产品可用。** Sora 能生成令人惊艳的 demo，但普通用户输入日常 Prompt 的结果远不如演示片段。生成内容可用率偏低——用户的期望被 demo 拉得很高，但实际产出远达不到。

**二问：商业模式不成立。** 日均 1500 万美元运营成本需要在消费者端回本——但消费者为一段 10 秒视频愿意付多少钱？$5？$20？即便按 $20/次计算也需要 75 万次日均付费生成。当前的视频生成需求远没到这个量级。

2026 年 4 月 Sora 消费者端正式关停。这给整个行业上了一课：**最好的技术不一定是成功的产品。** 同时也清空了一个赛道——给了 Runway 和国产方案快速崛起的机会。

## Runway：专业视频创作的瑞士军刀

Runway 是 Sora 退出后的最大受益者。但它和 Sora 走的是完全不同的路——Sora 追求"AI 自主生成"，Runway 追求"AI + 人工的专业创作工具"。

### Runway 的核心能力

**不只是生成，是完整的创作工作流**：

```
Runway 的一个完整工作流：
1. 文字描述 → 生成视频片段（5-10秒）
2. 关键帧控制 → 指定第一帧和最后一帧，AI填充中间帧
3. 运动笔刷 → 画出物体应该运动的路径和方向
4. 风格迁移 → 把实拍视频转成特定艺术风格
5. 超分辨率 → 把低分辨率视频升级到4K
```

这种"工具链"思维和 Sora 的"一镜到底"思维完全不同。Runway 认为专业创作者要的不是"AI 替我做好一切"，而是"AI 帮我更快地实现我的想法"。

### 和 Midjourney 的相似之处

Runway 在视频领域的定位很像 Midjourney 在图像领域的定位——不追求最先进的底层技术，但把产品体验和专业工具链做到了无可替代。

Midjourney 的画质不是靠什么独有算法，是靠理解"设计师想要什么样的输出"。Runway 同理——它的视频质量不是来自架构优势，是来自对创作者工作流的深入理解。

### Runway 的使用方式

Runway 提供 Web 端和 API。API 的定价按生成时长计费：

```
专业计划：$35/月
  625 credits/月（约 125 秒 1080p 视频生成）
  额外：$0.01/credit

一次典型的生成：10 秒 1080p ≈ 50 credits ≈ $0.50
```

## 国产视频生成：可灵与 Seedance 的双城记

### 可灵（快手）

可灵是 Sora 退出后国产视频生成的最大赢家。它的核心竞争力是**运动物理感**——人物动作和物体运动比同类产品更自然。

可灵的技术路线上有两个关键选择：
- 不做纯文生视频（text→video），大力投资图生视频（image→video）——因为从一张图开始，一致性更容易保证
- 支持视频延长和片段拼接——把多个 5-10 秒片段连成一段更长视频，这在实用层面解决了"5 秒太短"的核心痛点

可灵提供免费额度和付费套餐，对个人创作者友好——这和 Runway $35/月起跳的定价形成鲜明对比。

### Seedance（字节跳动）

Seedance 是字节跳动的视频生成模型，它和可灵的差异化在于**人物一致性**和**生态整合**：

- 同一人物的多段视频中保持外貌一致——这是 Seedance 的独特强项
- 与豆包 + 即梦 AI 深度联动——豆包写脚本→即梦/Seedance 生成视频→剪辑→抖音发布

Seedance 最大的优势不是画质，是**从创意到发布的完整工具体系**。对短视频创作者来说，Seedance + 即梦 + 豆包的组合几乎覆盖了制作一条视频所需的全部环节。

### 可灵和 Seedance 的对比

| 维度 | 可灵 | Seedance |
|------|------|----------|
| 技术强项 | 运动物理感/图生视频 | 人物一致性/文生视频 |
| 视频延长 | 支持 | 支持 |
| 免费额度 | 充足（每日数次） | 充足 |
| 生态 | 快手内容生态 | 字节内容生态（豆包/即梦/抖音） |
| 价格 | 免费+付费 | 免费+付费 |
| 最适用户 | 通用创作者 | 短视频创作为主的字节系用户 |

## 其他玩家：Veo、Pika、Luma

**Google Veo**。Gemini 体系内的视频生成，画质高但当前仅限部分用户和 Google Cloud 客户。与 Gemini 的视频理解能力打通——可以先生成视频，再用 Gemini 分析是否符合预期。

**Pika**。轻量级视频生成，创意玩法丰富——嘴唇同步、视频扩展、画布编辑。定位是"好玩的视频生成玩具"，不是专业创作工具。

**Luma Dream Machine**。3D 一致性有优势——生成的视频在多个视角间切换时不容易出现物体形变。这是因为 Luma 的核心技术积累在 3D 重建领域。

## 当前视频生成的技术天花板

2026 年中，AI 视频生成仍然有明显的天花板：

**时长瓶颈**。主力产品仍然是 5-10 秒的片段。通过拼接和延长可以拼出更长的视频，但编排和镜头切换需要人工剪辑。

**复杂动作形变**。跑步、跳舞、打斗——身体姿态剧烈变化时的形变问题仍然严重。手、脚在快速运动中容易畸变。

**镜头语言控制**。当前的 Prompt 控制粒度可以指定"内容"（一只猫），难以精确指定"镜头"（从低角度缓缓上摇的斯坦尼康跟拍）。Runway 的运动笔刷部分解决了这个问题，但远不够精确。

**声音同步**。视频生成模型通常只生成画面——口型、音效、背景音乐需要后期人工配音。AI 对口型技术已在快速进步（HeyGen、Synthesia 等），但和视频生成还没打通。

## 方案选型

| 你的场景 | 推荐方案 |
|---------|---------|
| 专业视频创作 | Runway Gen-4 |
| 中文日常视频 | 可灵（图生视频）/ Seedance（人物场景） |
| 短视频批量 | 即梦 AI + Seedance |

| 3D 视角视频 | Luma |
| Google 生态用户 | Veo（如有权限） |

## 总结

- 视频生成比图像难三个数量级：**计算量 ×120 帧 + 时序一致性 + 物理合理性**，单次成本差 3000 倍
- **Sora 的退场是一堂产品课**——技术最强不等于产品能活，消费者端的商业模式还没跑通
- **Runway 胜在工具链**——不是"AI 替你做"，而是"AI 帮你的创作工具"，专业创作者付费意愿最高
- **国产视频生成已具备实用价值**——可灵的物理感 + Seedance 的人物一致性 + 免费额度
- 当前天花板：5-10 秒时长、复杂动作形变、镜头控制粗糙、无声
